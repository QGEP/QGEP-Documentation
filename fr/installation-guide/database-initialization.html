1.2. Initialisation de la base de données¶

Vous pouvez utiliser pgAdmin afin d’accéder et gérer le serveur de base de données.

Note

En cliquant sur ce lien vous apprendrez à install pgAdmin

1.2.1. Procédure¶

Dans PgAdminIII

  • Se connecter au serveur de la base de données

  • Créer un nouveau group role (de préférences nommé qgep, mais libre à vous de choisir)

  • Créer un nouveau login role (qgepuser par exemple) et faites le devenir membre du groupe qgep

Il est possible de faire cela graphiquement ou plus simplement en exécutant le code suivant dans une requête SQL query :

CREATE GROUP qgep;
CREATE ROLE qgepuser LOGIN;
GRANT qgep TO qgepuser;

  • Créer une nouvelle base de données (de préférences nommée qgep, mais libre à vous de choisir)

    • Sélectionner cette base de données

  • Créer un nouveau schéma qgep

  • Ajouter les extensions hstore et postgis (si elles n’existent pas déjà) en exécutant la requête SQL query suivante:

    • CREATE EXTENSION hstore;
    • CREATE EXTENSION postgis;

1.2.1.1. Restore demo datamodel¶

Restore the latest data model that also includes demo data:

  • Télécharger les données de démonstration

    • https://github.com/QGEP/data/raw/demodata/qgep_demodata.backup

  • Clic-droit sur le schéma qgep

    • Cliquer sur Restore

    • Charger le fichier que vous venez de télécharger (qgep.backup)

    • Dans l’option #2: activez Clean before restore

    • Cliquer sur Restore

    • Vérifier que la dernière ligne du message windows soit: Exit Code 0

    • Cliquer sur Annuler

  • Clic-droit sur la base de données et cliquer sur Rafraîchir

  • Mettre à jour les privilèges pour la base de données

    • Clic-droit sur le schéma qgep

    • Properties -> Privileges Tab > Grant USAGE to group qgep. Vous pouvez aussi faire cela avec une requête SQl: ``GRANT USAGE ON SCHEMA qgep TO GROUP qgep;`

    • Clic droit sur le schéma et cliquer sur Grant Wizard …

    • Dans l’onglet selection, cliquez sur Check All

    • Privilèges

      • Groupe QGEP

      • Choisir ALL

1.2.1.2. Empty data model¶

You also have the option to restore the latest empty data model (no demo data).

  • Download the data model by going to https://github.com/QGEP/datamodel/releases and by downloading the latest template_db.dump.
  • In order to restore you can follow the steps from Restore demo datamodel, using the template data model instead of the demo data one.

1.2.1.3. Generate the data model¶

You can also generate the data model under Linux.

  • Download the data model:

    git clone https://github.com/QGEP/datamodel
cd datamodel && git submodule init && git submodule update

  • Setup the pg_service.conf file and add the pg_qgep service as in the Linux pg_service section.

  • Create the database:

    psql -U postgres -c 'CREATE DATABASE qgep;'

  • Run the generation script:

    ./scripts/dbsetup.sh

If you want to use a different SRID you need to use the -s option. For instance, run ./scripts/dbsetup.sh -s 2056 for the 2056 SRID.

If you already have a data model and you want to force the regeneration of the model you can also use the -f option: ./scripts/db_setup.sh -f.
