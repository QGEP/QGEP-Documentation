3.3. Échange de données INTERLIS

Il s’agit d’un guide sur la façon d’importer/exporter des données entre QGEP et INTERLIS 2 Transfer Format (xtf).

3.3.1. Général

Le plugin QGEP comprend une fonction expérimentale d’importation/exportation interlis. Il est actuellement capable d’importer et d’exporter vers le modèle SIA405 SIA405_ABWASSER_2015 (version 17.04.2018). Notez qu’actuellement, les exportations ne sont possibles qu’en allemand. Les exportations traduites sont sur la feuille de route, faites-nous savoir si vous êtes intéressé par cette fonctionnalité.

3.3.2. Prérequis

3.3.2.1. Java

Vous avez besoin de Java sur votre système.

Windows : download Max : download Linux : use you package manager

3.3.2.2. Bibliothèques Python

Vous avez besoin de sqlalchemy et geoalchemy2.

Pour installer : avant de lancer QGIS, ouvrez le Shell OSGeo4W, et tapez >>> pip3 install « sqlalchemy>=1.3.22 » « geoalchemy2>=0.9.0 ».

Si la commande échoue, essayez >>> pip install –upgrade pip

Si pip est introuvable, vérifiez que vous avez installé le paquet python3-pip en utilisant l’installateur réseau OSGeo4W (si vous avez installé la version autonome de QGIS, il devrait être inclus).

3.3.2.3. Modelbaker

Le plugin QGEP proposera d’installer le plugin Modelbaker automatiquement. Si cela n’a pas fonctionné, installez le plugin QGIS Modelbaker manuellement à partir du gestionnaire de plugins QGIS (version 6.4 ou plus récente).

3.3.2.4. Version QGEP

L’exportation ne prend en charge que le modèle de données QGEP le plus récent (1.5.6 au moment de la rédaction). Assurez-vous que votre modèle de données est entièrement mis à jour avant d’essayer d’importer/exporter.

3.3.3. Utilisation (GUI)

3.3.3.1. Activer le mode admin

Dans la boîte de dialogue plugin>QGEP>settings, sous l’onglet Developer options, assurez-vous que le Admin mode est activé. Redémarrez QGIS.

Vous devriez maintenant voir les nouveaux boutons import et export dans la barre d’outils QGEP.

3.3.3.2. Export

Pour exporter vos données QGEP, cliquez sur le bouton export. Le dialogue suivant apparaîtra.

Si vous avez une sélection active dans la couche des ouvrages et/ou des tronçons, vous pouvez choisir de limiter l’exportation à cette sélection. Ceci est particulièrement utile en combinaison avec les outils de sélection amont/aval.

Ensuite, confirmez le dialogue et choisissez où enregistrer le fichier .xtf.

Notez que pour des exportations importantes/complètes, la commande peut prendre beaucoup de temps.

Les exportations incluent une étape de validation utilisant ilivalidator, qui vous informera si l’exportation contient une erreur de validation.

3.3.3.3. Import

Pour importer des fichiers xtf, cliquez sur le bouton import et naviguez jusqu’au fichier .xtf. La boîte de dialogue suivante apparaît.

La partie gauche de ce dialogue liste tous les éléments qui vont être importés du fichier .xtf, permettant de revoir ce qui va être importé et de désélectionner les éléments que vous souhaitez ignorer. Elle montre également le statut de validation de chaque objet, indiquant si une action supplémentaire est nécessaire (INVALID) ou recommandée (WARNING) avant l’importation.

La partie droite de ce dialogue présente un formulaire spécifique au type d’élément sélectionné dans la liste, permettant d’adapter l’importation. Par exemple, il permet d’attacher des « examens » à leurs canalisations.

Une fois que vous êtes satisfait des options d’importation, confirmez la boîte de dialogue afin de conserver les changements dans votre base de données.

3.3.4. Utilisation (ligne de commande)

Pour une utilisation avancée, l’outil d’importation/exportation peut également être utilisé comme un outil de ligne de commande. Veuillez consulter le site https://github.com/QGEP/qgepqwat2ili/ pour obtenir de la documentation à ce sujet.
