2.2.2. Numériser les structures assainissement¶

2.2.2.1. Généralités¶

QGEP has a wizard to correctly collect manholes and special structures. see the wizard chapter.

Sélectionnez le bouton Assistant puis cliquez sur le bouton Commencer la saisie de données et choisir Ouvrage assainissement dans le menu déroulant.

2.2.2.2. Numérisation¶

Le curseur passe alors en mode numérisation et il est possible de sélectionner l’emplacement du nouvel élément.

Le formulaire vw_qgep_structure_assainissement s’ouvre ensuite et vous pouvez commencer la saisie de données dans l’onglet Général:

Sélectionnez le ws_type désiré (“regard” est sélectionné par défaut):

  • Regard

  • structure_speciale

  • point_deversement

  • installation_infiltration

Selon le ws_type, vous aurez différents champs et onglets dans le formulaire.

Ajoutez ensuite l’identifiant (il s’agit de l’attribut qui sera affiché sur la carte)

Note

Si vous n’entrez pas d’identifiant, QGEP entrera l’obj_id en tant qu’identifiant (vous pouvez le modifier plus tard). Par défaut, l’identifiant de la structure assainissement est également l’identifiant du couvercle et du noeud assainissement.

Ajoutez d’autres attribus dans l’onglet Général. Vous pouvez également ajouter des attributs dans les autres onglets (Couvercle, Structure assainissement, Regard, Noeud assainissement).

Note

Le concept de l’onglet Général est que, dans le processus normal de numérisation (95% des regards), l’opérateur n’a pas besoin de changer d’onglet pour renseigner les attributs nécessaires.

Attention

Vous ne pouvez pas ajouter de nouveaux entrées dans les onglets Couvercles, Noeuds assainissement et Maintenance, car vous devez sauvegarder au préalable l’enregistrement de la structure assainissement dans la base de données avant de pouvoir ajouter de nouvelles entrées. Pour la même raison, vous ne pouvez pas utiliser Actions. Sauvegardez d’abord l’entrée.

Cliquez OK pour fermer le formulaire

Sauvegardez les informations de la couche en cliquant sur « Stop Data Entry » dans l’assistant (« wizard »)

Pour rééditer l’objet, passez en mode d’édition et cliquez sur l’objet à éditer avec le curseur en mode « Identifier les entités ». Si le mode d’édition n’est pas activé, vous pourrez seulement consulter les informations relatives à l’objet.

Pour plus d’informations au sujet de l’édition de données, voir le chapitre Modification de données existantes.

2.2.2.3. Autres classes et attributs¶

Lorsqu’une structure_assainissement est numérisée, une série d’opérations se déroulent en arrière-plan dans la base de données QGIS:

  1. Un nouvel objet est ajouté dans la classe ouvrage_assainissement

  2. Un nouvel objet sera crée et lié dans les sous-classes respectives [point_deversement, installation_infiltration, regard, ouvrage_spécial]

  3. un nouvel objet couvercle est créé et lié à l’ouvrage assainissement

  4. un nouvel objet noeud assainissement est généré dans la couche éléments du réseau assainissement et sa sous-classe noeuds assainissement

To add additional objects such as structure parts (besides covers there are access aids, back flow prevention etc) you have to save first and then edit the point object (see the Modification de données existantes chapter).

Note

Pour ajouter un second couvercle ou un second noeud assainissement à un ouvrage assainissement, voir le chapitre Modification de données existantes.

2.2.2.4. Synchronisation de la géométrie¶

La géométrie de la nouvelle entité définit la géométrie des tables connectées comme “couvercle” et “noeud assainissement”. La valeur Z de la géométrie peut être fixée en définissant les valeurs de cotes. Cela signifie que lorsque la cote du couvercle co_level est entrée, cette valeur est adaptée à la valeur Z de la géométrie du couvercle. La cote du radier du noeud assainissement wn_bottom_level définit la valeur Z de la géométrie du noeud assainissement. La géométrie de l’entité noeud assainissement correspond à celle du couvercle. Cela signifie que la géométrie Z de l’entité numérisée est en fait la valeur entrée dans co_level.

Note

If a cover level changes, the Z value of the cover’s geometry will be ajusted. When the geometry changes, the level is adjusted as well. If both values change, the level takes precedence. On an insert it’s like when both value change. Means the cover’s geometry is set according to the cover level and if it’s NULL, the Z value is set to NaN. The same situation is on editing the wastewater node directly. On this main layer wastewater structure, the only way to set the Z value of the geometry is to set co_level for cover and wn_bottom_level for wastewater node.
