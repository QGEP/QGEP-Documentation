2.2.5. Kanäle Digitalisieren¶

2.2.5.1. Allgemeines¶

QGEP hat einen Assistenten, um Kanäle korrekt zu erfassen und diese mit Schächten respektive Abwasserknoten oder Haltungen zu verbinden (und dabei die Abwasserknoten- und Haltungs-Topologie aufzubauen).

Achtung

Im qgep plugin Version 1.1.4 ist eine Fehler. Vgl. QGEP Issue #443 und #451.

Achtung

Starten Sie die Digitalisierung von Haltungen/Kanälen immer in Fliessrichtung des Kanales, indem sie mit dem from manhole node starten und mit dem to manhole node aufhören.

2.2.5.2. Digitalisieren¶

  • Im Digitalisierungsmodus greift der Cursor automatisch auf die nächstgelegene Abdeckung oder Haltung zu. Mit einem Linksklick beginnen Sie eine Linie zu zeichnen.

  • Mit weiteren Klicks irgendwo können Sie die dazwischenliegenden Punkte des Haltungsverlaufs definieren. Sie können auch direkt einen anderen Kanalschacht anwählen, um einen geraden Kanal zu zeichnen.

  • Sie können das Digitalisieren der Linie mit einem Rechts Klick beenden. Es erscheint das vw_qgep_reach (Haltungs-)Formular.

Bemerkung

Behalten Sie im Kopf, dass der Endpunkt der Linie der Punkt ist, an welchem Sie zuletzt Links geklickt haben. Deshalb benötigen Sie, für die Digitalisierung einer einfachen Linie mit 2 Punkten, zwei Links Klicks um die Linie zu digitalisieren und einen Rechts Klick um die Digitalisierung der Linie zu beenden.

  • Fügen Sie ander Attribute im Reiter General hinzu.

Bemerkung

Das Schnappen auf Abwasserknoten oder andere Haltungen kann in den Feldern rp_from/to_fk_wastewater_networkelement kontrolliert werden.

  • Für den Profiltyp erhalten Sie eine Liste von vordefinierten Profilen. Sie können diese in der od_pipe_profile Tabelle bearbeiten.

  • Wenn Sie fertig sind klicken Sie den OK Knopf

  • Speichern Sie die Informationen des Layers, indem Sie den Data entry Wizard beenden

  • Sie können Ihr Punktobjekt bearbeiten indem Sie den Bearbeitungsmodus auswählen und dann mit dem Info Kursor auf das Objekt klicken, welches Sie bearbeiten möchten.

  • Wenn Sie den Bearbeitungsmodus nicht auswählen, können Sie nur die existierenden Daten anschauen.

  • Für detailliertere Informationen über das Bearbeiten lesen die das Kapitel Bearbeiten von bestehenden Daten.

2.2.5.3. Weitere Attribute und Klassen¶

Wenn ein Linienobjekt digitalisiert wird, finden eine Reihe von Schritten im Hintergrund in der QGEP Datenbank statt:

  • Ein neues Objekt wird in der Abwasserbauwerk-Tabelle hinzugefügt (od_wastewater_structure)

  • Ein neues Objekt wird hinzugefügt und verknüpft in der Kanal Unterklasse (od_channel)

  • zwei neue Haltungspunktobjekte werden hinzugefügt und verknüpft mit der Haltung (rp_from_node, rp_to_node)

  • ein neues Haltungsobjekt wird in der Abwassernetzelement Tabelle (od_wastewater_networkelement) und dessen Unterklasse Haltung (od_reach) generiert.

2.2.5.4. Geometrieabgleichung¶

Start- und Endpunkt der Geometrie des Features bestimmen die Geometrie der Reachpoints. Die Höhe (Z Wert der Geometrie) des Start- und Endpunktes des hinzugefügten Features wird über die Level-Werte rp_from_level und rp_to_level definiert. Das heisst, der Z Wert der Geometrie des Reachpoints wird ebenso über den Level-Wert gesetzt. Diese Werte können über das “gefangene” Feature gesetzt werden:

Die “gefangenen” Features 9405 und 9390 haben die Höhe ensprechend den Coverlevels: 424.090 und 422.900. Diese Werte werden in die Attribute rp_from_level und rp_to_level eingetragen, wo sie vom Benutzer editiert werden könnten. Wenn der Benutzer diese Werte ändert, wird die Geometrie des digitalisierten Features und seinen Reachpoints ebenfalls geändert.

Bemerkung

Wenn das Level eines Reachpoints ändert, ändert sich auch der Z Wert seiner Geometrie und somit auch die Geometrie des Start- und Endpunktes der Reach. Wenn der Z Wert der Geometrie der Reach ändert, ändert auch die Geometrie und das Level der Reachpoints. Wenn beide Werte gleichzeitig ändern, hat das Level Vorrang. Beim Einfügen ist es wie wenn beide Werte ändern. Also wird die Geometrie der Reach gemäss der Level der Reachpoints gesetzt und falls diese NULL sind, werden die Z Werte des Start- und Endpunktes der Reach auf NaN gesetzt.
